## How Can I Contribute?

### Reporting Bugs

**Before Submitting:**
- Check if the bug has already been reported in [Issues](../../issues)
- Use the latest version of Vault Keeper

**When Submitting:**
- Use a clear, descriptive title
- Describe the exact steps to reproduce
- Provide your setup details (Obsidian version, Claude version, OS)
- Include relevant journal template sections if applicable
- Share any error messages or unexpected behavior

### Suggesting Enhancements

We love feature suggestions! Please:
- Use a clear, descriptive title
- Explain the use case (why this would be valuable)
- Describe the expected behavior
- Consider if it fits the core philosophy (accountability, not automation)

### Template Contributions

Have a great journal template for a new focus area? Submit it!

**Template Requirements:**
- Follows existing template structure (see TEMPLATES.md)
- Includes clear tracking metrics
- Tested with actual journaling (at least 7 days)
- Includes example entries

### Pattern Library Examples

Share anonymized patterns Claude discovered about you:
- Helps others see what's possible
- Improves pattern recognition guidelines
- Builds community knowledge

**Submit to:** `examples/patterns/` (anonymize all personal data)

## Development Process

1. **Fork the repo**
2. **Create a feature branch** (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Test thoroughly** (ideally with real vault usage)
5. **Commit** (`git commit -m 'Add amazing feature'`)
6. **Push** (`git push origin feature/amazing-feature`)
7. **Open a Pull Request**

## Pull Request Guidelines

- Clear description of changes
- Reference any related issues
- Include before/after examples if applicable
- Update documentation if needed
- Test with actual Obsidian vault + Claude

## Code of Conduct

### Our Standards

**Positive behavior:**
- Using welcoming, inclusive language
- Respecting differing viewpoints
- Accepting constructive criticism
- Focusing on what's best for the community

**Unacceptable behavior:**
- Harassment or discriminatory language
- Trolling or insulting comments
- Publishing others' private information
- Other conduct inappropriate in a professional setting

### Enforcement

Project maintainers will:
- Clarify standards of acceptable behavior
- Take appropriate corrective action for unacceptable behavior
- Remove, edit, or reject contributions that don't align with this Code
